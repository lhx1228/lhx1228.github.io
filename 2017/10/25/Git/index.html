之后执行text.txt文件的版本回退操作，回退完成后我们发现，123.txt文件消失了！！！

这是为什么呢？我想这应该与Vmware的快照功能差不多，如果选择回退，就会将整个工作区全都回退到之前的那个状态，而不仅仅回退某个文件的某个内容.

也就是说，Git跟踪并管理的是修改，而非文件。

那什么是修改？比如你新增了一行，这就是一个修改，删除了一行，也是一个修改，更改了某些字符，也是一个修改，删了一些又加了一些，也是一个修改，甚至创建一个新文件，也算一个修改。

我们回顾一下过程：

第一次修改->git add->第二次修改->git commit

你看，我们前面讲了，Git管理的是修改，当你用git add命令后，在工作区的第一次修改被放入暂存区，准备提交，但是，在工作区的第二次修改并没有放入暂存区，所以，git commit只负责把暂存区的修改提交了，也就是第一次的修改被提交了，第二次的修改不会被提交。
