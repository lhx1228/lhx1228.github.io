正则表达式

发表于 2017-11-06 | 分类于 正则表达式 | 阅读次数

正则表达式由一些普通字符和一些元字符组成。
–《正则表达式之道》

普通字符包括大小写的字母和数字，元字符则具有特殊的含义.

正则表达式

正则表达式基本语法

符号

  • \：’\’一般用来做字符标记符，如：’\\n’表示’\n’;向后引用，如：’(\d)\1’,’\1’表示引用’\d’匹配到的数字；八进制转义符。

  • ^：’^’一般用来标识匹配字符串的开始位置，例如：^[0-9]表示 匹配头为数字的字符串；’^’还可以用来标识反，例如:[^0-9]表示匹配除了0-9的其他字符。

  • $：’$’的作用和’^’的作用相反，即匹配字符串的结束位置。

  • {n}*：’{n}’表示匹配n次，例如：\d{2}表示匹配两次[0-9]。而’{n,}’表示匹配>=n次， 例如：{0,}表示匹配大于等于 1次，等同于’*‘;{1,}表示匹配大于等于 1次，等同于’+’;’{n,m}’表示匹配最少n次最多m次，例如：{0,1}表示匹配最多一次，等同于’?’。

  • ‘.’：’.’表示匹配除’\r’’\n’以外的任意字符。

字符组简写式

  • ‘\d’：’\d’表示匹配任意数字字符。等价于[0-9]。

  • ‘\D’：’\D’与’\d’相反，表示匹配非数字字符。等价于[^0-9]。

  • ‘\w’：’\w’表示匹配包括下划线的任何单词字符。

  • ‘\W’：’\W’与’\w’相反，表示匹配包括下划线的任何单词字符。

  • ‘\b’：’\b’匹配一个单词边界，也就是指单词和空格间的位置。例如：er\b可以匹配”never”中的”er”，但不可以匹配”verb”中的”er”。

  • ‘\B’：’\B’匹配非单词边界。er\b可以匹配”verb”中的”er”，但不可以匹配”never”中的”er”。

  • ‘\s’：’\s’匹配任何不可见字符，包括空格、制表符、换页符等等。等价于[ \f\n\r\t\v]。

  • ‘\S’：’S’匹配任何可见字符。等价于[^ \f\n\r\t\v]。

  • ‘\cx’：’\cx’匹配由x指明的控制字符。例如，’\cM’匹配一个Control-M或回车符。x的值必须为A-Z或a-z之一，否则，将c视为一个原义的“c”字符。

  • ‘\<’、’\>’：’\<’、’>‘匹配词（word）的开始（\<）和结束（>）。例如正则表达式\能够匹配字符串”for the wise”中的”the”，但是不能匹配字符串”otherwise”中的”the”。

经过以上总结，我们发现\D和\w似乎有点相似，那它们有什么差别呢？

\w表示匹配单词字符和下划线’‘，单词包括数组和字母；而\D表示匹配非数字字符，即[^0-9]，它会匹配字母和各种符号，不仅仅是下划线’‘。

实例

以下实例均为自己笔者所写，若有更好的方法欢迎在评论区留言~

  • 匹配手机号码：“^1[3|4|5|7|8]\d{9}$”

  • 匹配身份证号：“\d{6}1[0-9]{10,11}[0-9xX]”

  • 匹配email地址：“\w+@\w+.\w+”

  • 匹配URL：“^(http\:\/\/|https\:\/\/)(\w+[.,\/,?,=,%,&]*)*$”

  • 匹配IP地址：“^(\d{1,3}.){3}[1-9]{1,3}$”

a命令与i命令

a即append,i即insert，这两个命令都用来添加行。

下面看例子：

sed "1 i This is a test" test.txt
这个命令在第一行前(insert)插入一行。

sed "$ a This is a test" test.txt
这个命令在最后一行追加(append)一行。

当然除了可以在某一行进行添加操作外，我们还可以使用匹配来进行添加：

sed "/\([0-9]\{1,3\}\.\)\{3\}[0-9]\{1,3\}/a 123456" pets.txt

上面的命令可以在匹配到IP地址后，在其后一行添加123456。
